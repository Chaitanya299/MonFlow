Foreground Persistence Approaches

Ensuring the Monflo Listener stays alive

Approach 1: Shared Lifecycle Bridge (Recommended)

The NotificationListenerService itself handles the foreground promotion. When React Native toggles the switch, it calls a bridge method that starts/stops the service with a START_STICKY flag.

  • Cleaner architecture: tracking logic and persistence live in one class
  • Simpler auto-start on boot logic

Approach 2: Separate Service Manager

A separate ForegroundService.kt manages the notification and keep-alive state. It "Monitors" the NotificationListenerService and restarts it if it crashes.

  • Over-engineered for a v1 prototype
  • More complex communication between two native services

My Recommendation

Approach 1: Shared Lifecycle Bridge is the most "Industry Level" path for a focused utility app. It minimizes the surface area for bugs and provides a direct link between the UI toggle and the system service state.
