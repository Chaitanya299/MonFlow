Hardware-Backed Security Architecture

Comparing current prototype vs. production-grade vault

Current: Prototype (Low Security)
Key Storage

Hardcoded: "prototype_key"

Flow
[Service]
   |
   +--> Database(passphrase="prototype_key")

Risks

  • Passphrase stored in APK
  • Vulnerable to static analysis
Proposed: Vault-Grade (High Security)
Key Storage

Android Keystore: Hardware-protected

Flow
[Service]
   |
   +--> KeystoreManager.getOrGenerateKey()
   |      |
   |      +--> [TEE / Secure Element]
   |
   +--> Database(passphrase=derived_key)

Benefits

  • Key never leaves hardware
  • Encrypted even on rooted devices

Clarifying Question

To implement this, we need to decide how to handle the First Launch. If no key exists in the Keystore, should we:

A

Auto-Generate

App generates a random 256-bit key on first run silently.

B

User-Passphrase

User creates a PIN/Password, which is then used to wrap the hardware key.
