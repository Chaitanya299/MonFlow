Biometric & Background Flow

Deciding how "Deep" the locking goes

Option A: Transparent Tracking (Recommended)
Background

Service uses Keystore key without user interaction to save alerts.

UI Access

App requires Biometrics to see the charts or ledger.

Pros

  • Background tracking always works
  • No missed transactions
Option B: Cold Vault (Max Privacy)
Background

Database is locked. Alerts are buffered in RAM until user unlocks.

UI Access

User must authenticate with Fingerprint/Face to start the app.

Risks

  • Data lost if phone reboots before unlock
  • RAM buffer limit risk

Clarifying Question

Should the tracking service have background access to the vault?

A

Transparent Tracking

Track in background silently; Biometric lock for UI only.

B

Cold Vault

DB remains locked until the user physically unlocks the app.
